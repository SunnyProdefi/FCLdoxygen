\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename S\+\_\+$>$\newline
class fcl\+::\+Convex$<$ S\+\_\+ $>$}

A convex polytope. 

The Convex class represents a subset of general meshes\+: convex meshes. The class represents its underlying mesh quite simply\+: an ordered list of vertices, {\ttfamily V = \mbox{[}v₀, v₁, ..., vₙ₋₁\mbox{]}}, and an ordered list of faces, {\ttfamily F = \mbox{[}f₀, f₁, ..., fₘ₋₁\mbox{]}}, built on those vertices (via vertex {\itshape indices}).

A mesh is only compatible with Convex if it satisfies the following properties\+:
\begin{DoxyItemize}
\item Each face, fᵢ, must be planar and completely lies on a supporting plane πᵢ. The ordered sets {\ttfamily F = \mbox{[}f₀, f₁, ..., fₙ₋₁\mbox{]}} and {\ttfamily Π = \mbox{[}π₀, π₁, ..., πₙ₋₁\mbox{]}} are defined such that face fᵢ has supporting plane πᵢ.
\item A face fᵢ is defined by an ordered list of vertex indices (e.\+g., {\ttfamily fᵢ = \mbox{[}iᵢ₀, iᵢ₁, ..., iᵢₖ₋₁\mbox{]}} (for a face with k vertices and k edges). The {\itshape ordering} of the vertex indices must visit the face\textquotesingle{}s vertices in a {\itshape counter-\/clockwise} order when viewed from outside the mesh and the vertices must all lie on the face\textquotesingle{}s supporting plane.
\item Define functions {\ttfamily πᵢ(x)} which report the signed distance from point {\ttfamily x} to plane {\ttfamily πᵢ}. To be convex, it must be true that {\ttfamily π(v) ≤ 0, ∀ π ∈ Π ∧ v ∈ V}.
\end{DoxyItemize}

If those requirements are satisfied, for a given convex region, the mesh with the smallest number of faces {\itshape may} have non-\/triangular faces. In fact, they could be polygons with an arbitrary number of edges/vertices. The Convex class supports compact representation. But it {\itshape also} supports representations that decompose a large n-\/gon into a set of smaller polygons or triangles. In this case, each of the triangles supporting planes would be the same and the requirements listed above would still be satisfied.

Gets the representation of the {\itshape faces} of the convex hull. 

The array is the concatenation of an integer-\/based representations of each face. A single face is encoded as a sub-\/array of ints where the first int is the {\itshape number} n of vertices in the face, and the following n values are ordered indices into {\ttfamily vertices} which visit the vertex values in a {\itshape counter-\/clockwise} order (viewed from the outside).

For a well-\/formed face {\ttfamily f} consisting of indices \mbox{[}i₀, i₁, ..., iₘ₋₁\mbox{]}, it should be the case that\+:

{\ttfamily eⱼ × eⱼ₊₁ · n̂ₚ = $\vert$eⱼ × eⱼ₊₁$\vert$, ∀ 0 ≤ j $<$ m, j ∈ ℤ}, where {\ttfamily n̂ₚ} is the normal for plane {\ttfamily π} on which face {\ttfamily f} lies. {\ttfamily eⱼ = vertices\mbox{[}iⱼ\mbox{]} -\/ vertices\mbox{[}iⱼ₋₁\mbox{]}} is the edge vector of face {\ttfamily f} defined by adjacent vertex indices at jᵗʰ vertex (wrapping around such that {\ttfamily j -\/ 1 = m -\/ 1} for {\ttfamily j = 0}).

Satisfying this condition implies the following\+:
\begin{DoxyEnumerate}
\item vertices are not coincident and
\item the indices of the face correspond to a proper counter-\/clockwise ordering. 
\end{DoxyEnumerate}\mbox{\Hypertarget{classfcl_1_1Convex_a2c6c2bfa3b5380e374ad6ae7bf8e8972}\label{classfcl_1_1Convex_a2c6c2bfa3b5380e374ad6ae7bf8e8972}}
